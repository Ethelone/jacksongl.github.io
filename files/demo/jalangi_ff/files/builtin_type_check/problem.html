Runtime Type Check for Built-in Functions

JavaScript is a dynamically typed language. While is OK to assign different types of values to the same variable or defined functions with arbitrary parameters during the execution, most JavaScript built-in functions (e.g., String.prototype.indexOf etc.) are defined with valid parameter type. However, it is generally hard to precisely infer the type of a value via static analysis. Building a runtime type checker for those standard built-in functions can hopefully help pinpoint potential problems.
For illustration, this demo analysis only checks String.prototype.localeCompare function, a relatively more complete runtime checker is available as analyses in the DLint probject.
