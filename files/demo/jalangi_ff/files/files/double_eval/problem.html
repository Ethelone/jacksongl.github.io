Detect Double Evaluation

In JavaScript standard, there are four functions that allow you to take as an argument a string code and evaluate it from within the running code: eval, Function, setTimeout and setInterval. But when you uses those functions with a string code argument, you incur a double evaluation penalty: it has to create a new interpreter/compiler instance and process the passed in string code.
Based on common code practise, there is no need to use eval() or Function() in most of the time. It is also best to to pass in a function as the first argument (instead of a string code) when using setTimeout() and setInterval()
